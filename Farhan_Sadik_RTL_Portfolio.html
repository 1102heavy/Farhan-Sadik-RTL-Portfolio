01 — Flagship Project
AXI-Stream Matrix Multiplier Coprocessor
Hardware acceleration on Xilinx Kria KV260 — from RTL through HLS, with full verification and performance benchmarking.

Architecture & Implementation

EE4218 · Kria KV260 · 2026
Verilog Vitis HLS Vivado IP Integrator C (Vitis) AXI4-Stream FSM

Designed a streaming matrix-multiplication coprocessor with S_AXIS input and M_AXIS output, implementing fixed-point RES = A·B / 256 arithmetic. The core uses an FSM with dedicated A, B, and Result RAMs, Start/Done handshaking, and supports continuous operation without external reset between jobs. Integrated into a Vivado platform with ARM Cortex-A53 host, AXI Stream FIFO, and timer peripherals.

  • Built self-checking verification with .mem stimulus/expected vectors; validated TVALID/TREADY backpressure and TLAST framing across behavioural and post-synthesis simulation
  • Explored HLS optimisations: pipeline, unroll, array partition, dataflow — evaluated resource vs. performance trade-offs against HDL implementation
  • Refactored FSM control to minimise overhead cycles between transfers, improving accelerator utilisation
  • Wrote Vitis C benchmarking code to stream CSV matrices, measure AXI Timer cycle counts, and compare SW baseline vs. HDL vs. HLS accelerator

03 — Processor Design
Pipelined RISC-V CPU on Artix-7 FPGA
Full datapath + control in Verilog with hazard handling, deployed on Nexys A7 hardware.

5-Stage Pipeline with Forwarding & Stall Logic

CG3207 · Nexys A7 · 2025
Verilog Artix-7 FPGA RISC-V ISA Pipeline Hazard Unit

Designed a synthesisable pipelined RISC-V CPU implementing the core integer ISA with a 5-stage pipeline (IF/ID/EX/MEM/WB), complete control unit, and hazard handling through data forwarding and pipeline stalling.

  • Implemented data forwarding paths and stall logic to maintain ISA-correct execution under all hazard conditions
  • Verified with self-checking simulation testbenches and directed instruction sequences; debugged with waveform analysis
  • Applied RTL best practices: clean module interfaces, clock/reset discipline, timing-aware design from simulation to on-board bring-up
  • Used on-board LEDs, switches, seven-segment displays and UART for hardware debugging and verification
  • Wrote C/assembly micro-benchmarks to measure cycles, CPI, and latency — validated forwarding/stall behaviour against expected performance
